Hooks are user-defined shell commands, HTTP endpoints, or LLM prompts that execute automatically at specific points in Claude Code’s lifecycle. Use this reference to look up event schemas, configuration options, JSON input/output formats, and advanced features like async hooks, HTTP hooks, and MCP tool hooks. If you’re setting up hooks for the first time, start with the guide instead. ​ Hook lifecycle Hooks fire at specific points during a Claude Code session. When an event fires and a matcher matches, Claude Code passes JSON context about the event to your hook handler. For command hooks, input arrives on stdin. For HTTP hooks, it arrives as the POST request body. Your handler can then inspect the input, take action, and optionally return a decision. Some events fire once per session, while others fire repeatedly inside the agentic loop: The table below summarizes when each event fires. The Hook events section documents the full input schema and decision control options for each one. The matcher narrows to Bash tool calls and the if condition narrows further to commands starting with rm , so block-rm.sh only spawns when both filters match: { &quot;hooks&quot; : { &quot;PreToolUse&quot; : [ { &quot;matcher&quot; : &quot;Bash&quot; , &quot;hooks&quot; : [ { &quot;type&quot; : &quot;command&quot; , &quot;if&quot; : &quot;Bash(rm *)&quot; , &quot;command&quot; : &quot; \&quot; $CLAUDE_PROJECT_DIR \&quot; /.claude/hooks/block-rm.sh&quot; } ] } ] } } The script reads the JSON input from stdin, extracts the command, and returns a permissionDecision of &quot;deny&quot; if it contains rm -rf : #!/bin/bash # .claude/hooks/block-rm.sh COMMAND = $( jq -r &#x27;.tool_input.command&#x27; ) if echo &quot; $COMMAND &quot; | grep -q &#x27;rm -rf&#x27; ; then jq -n &#x27;{ hookSpecificOutput: { hookEventName: &quot;PreToolUse&quot;, permissionDecision: &quot;deny&quot;, permissionDecisionReason: &quot;Destructive command blocked by hook&quot; } }&#x27; else exit 0 # allow the command fi Now suppose Claude Code decides to run Bash &quot;rm -rf /tmp/build&quot; . Here’s what happens: 1 Event fires The PreToolUse event fires. Claude Code sends the tool input as JSON on stdin to the hook: { &quot;tool_name&quot; : &quot;Bash&quot; , &quot;tool_input&quot; : { &quot;command&quot; : &quot;rm -rf /tmp/build&quot; }, ... } 2 Matcher checks The matcher &quot;Bash&quot; matches the tool name, so this hook group activates. If you omit the matcher or use &quot;*&quot; , the group activates on every occurrence of the event. 3 If condition checks The if condition &quot;Bash(rm *)&quot; matches because the command starts with rm , so this handler spawns.
